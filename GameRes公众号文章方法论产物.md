## 3. 方法论总纲

从整个公众号专辑中提炼出的核心判断：

| 方向 | 核心方法论 |
|---|---|
| 立项 | 框架 → 体验 → 场景；不要只追热点，要验证用户需求和团队最长板 |
| 关卡 | 关卡不是场景，而是塑造玩家行为的空间 |
| 数值 | 先有体验需求，再有数值；数值是调节器和显示器 |
| 叙事 | 游戏叙事要让玩家亲手参与，不只是阅读剧情 |
| 角色 | 角色吸引力来自价值与可得性的平衡 |
| 系统 | 系统设计要形成循环、反馈和长期目标 |
| 商业化 | 商品价值来自玩家未满足需求的痛苦强度 |
| 社交 | 社交本质是需求与被需求关系 |
| 独立开发 | 控制范围，尽早验证，不要陷入照抄成功案例的局部最优 |

## 4. 可直接复用的策划检查清单

### 4.1 立项前

- [ ] 核心循环能否一句话说清？
- [ ] 玩家 30 秒、30 分钟、30 小时分别获得什么体验？
- [ ] 产品解决的是创新、完善、细分还是题材机会？
- [ ] 团队最长板是否匹配核心体验？
- [ ] 是否只是把热门机制拼在一起？
- [ ] Demo 要验证的最大风险是什么？

### 4.2 关卡/流程设计

- [ ] 这个空间要塑造玩家什么行为？
- [ ] 是否有清晰起因、经过、结果？
- [ ] 是否有至少一个峰值体验点？
- [ ] 玩家迷路时是否有面包屑引导？
- [ ] 美术是否服务玩法和引导？
- [ ] 重复内容是否被折叠或跳过？

### 4.3 叙事设计

- [ ] 玩家是在读故事，还是通过动作推进故事？
- [ ] 每个关键真相是否由玩家亲手发现？
- [ ] 碎片化叙事是否降低了记忆成本？
- [ ] 环境、物件、UI 是否承担叙事？
- [ ] 结局是否是价值表达，而不是简单好坏题？

### 4.4 数值/系统

- [ ] 自变量是什么？例如战斗时长、击杀刀数、成长速度
- [ ] 公式服务的体验是什么？
- [ ] 是否考虑最大值、最小值和极端情况？
- [ ] 反馈是否足够清晰？
- [ ] 上线后准备观察哪些数据？

### 4.5 NPC/角色

- [ ] NPC 是否有知识边界？
- [ ] NPC 是否会因玩家证据改变状态？
- [ ] NPC 行为是否可观察、可中断、可解读？
- [ ] 角色价值是否清晰？
- [ ] 是否避免只用台词解释人物？

### 4.6 文本量控制

- [ ] 单段文本是否过长？
- [ ] 文本是否能转化为卡片、物件、票据、标记？
- [ ] 阅读是否有明确收益？
- [ ] 玩家跳过文本后是否仍能理解核心目标？
