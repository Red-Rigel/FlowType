# 02 核心设计框架

## 一、MDA 框架

**Mechanics（机制）→ Dynamics（动态）→ Aesthetics（美学/体验）**

| 层级 | 定义 | 设计者视角 | 玩家视角 |
|------|------|------------|----------|
| M 机制 | 规则、数据、算法 | 直接设计 | 不可见 |
| D 动态 | 运行时涌现的行为 | 间接设计 | 可感知策略 |
| A 美学 | 情感体验 | 设计目标 | 乐趣来源 |

### 1.1 三种设计切入角度

| 切入 | 优点 | 局限 | 适用 |
|------|------|------|------|
| **从美学(A)入手** | 情感共鸣强、易传播 | 易忽略玩法、资源浪费 | 独立游戏、叙事向 |
| **从机制(M)入手** | 把控细节、适合纸面模型 | 难把控全局、易重复造轮子 | 独立、策略游戏 |
| **从动态(D)入手** | 面向策略体验、适合改良 | 需原型、非从零创新 | 商业游戏迭代 |

### 1.2 玩法设计路径（机制派）

```
建立基础规则 → 丰富/完善规则 → 按体验目标裁剪
```

**案例：猜拳玩法扩展**
- 基础：石头剪刀布
- 深度化：不同手势伤害/范围不同
- 广度化：手势可组合连招
- 裁剪：去掉破坏节奏的组合

## 二、巴图玩家分类 → 策划能力线

### 2.1 四象限

```
              Acting（竞争）
                  │
    Killer ───────┼─────── Achiever
                  │
  Player ─────────┼───────── World
                  │
  Socializer ─────┼─────── Explorer
                  │
            Interacting（互动）
```

| 类型 | 乐趣来源 | 对应策划线 |
|------|----------|------------|
| Killer | PVP 竞争获胜 | **规则构建** |
| Achiever | 挑战系统目标 | **规则构建** |
| Socializer | 与人交流 | **目标供给** |
| Explorer | 发现未知内容 | **目标供给** |

### 2.2 规则构建者

**任务：** 构建核心玩法

**基本玩法 = 基本问题 + 基本解法**

| 概念 | 说明 | 射击游戏例 |
|------|------|------------|
| 基本问题 | 不可再分的游戏目标 | 消灭敌人、躲避伤害、到达位置 |
| 基本解法 | 对应一种基本操作 | 射击、闪避、移动 |
| 能力维度 | 玩家可控制的变量 | 攻击/防御/移动 |

**进阶玩法：**
- **深度化**：同维度内差异化（A弹/B弹、不同职业）
- **广度化**：多维度复合（边打边躲、打+放技能）

> ⚠️ 问题与解法必须成对出现，单独增加任一方都会造成浪费和误导。

**乐趣循环（基本玩法）：**
```
学习规则 → 操控 → 反馈 → 成长认知 → 挑战达成 → 激励 → 学习新规则
```

**乐趣循环（进阶玩法）：**
```
系统设定挑战 → 选择/组合解法 → 试验 → 反馈 → 改进 → 逼近最优解 → 大量成就感
```

### 2.3 目标供给者

**任务：** 创造外生目标，驱动长期投入

| 目标类型 | 特征 | 乐趣循环 | 例子 |
|----------|------|----------|------|
| 固定回报 | 100%概率固定奖励 | 量变→质变→激励 | 升级、任务、收集 |
| 随机回报 | 概率性不确定奖励 | 摇奖→开奖→累积 | 掉落、抽奖、强化 |
| 复合回报 | 固定+随机叠加 | 两者叠加 | 锻造、套装 |

**现代游戏标配三要素（口袋妖怪）：**
- 收集
- 强化
- 交换

## 三、玩法维度分析法

### 3.1 定义

- **玩法元素**：一次完整的 观察→思考→决策→操作 循环
- **玩法维度**：共享同一决策循环的大量玩法元素集合

### 3.2 反拆步骤

1. 游玩并**记录所有思考与抉择**
2. 按观察要素、行为类型归类
3. 划分玩法维度
4. 识别维度间的耦合关系
5. 针对目标用户做维度取舍

### 3.3 改良而非克隆的关键

- 合理拆分元素与维度
- 有取舍地保留优点、剔除缺点
- 避免「纳米级克隆」—— 无法分析就只能全盘复制

## 四、体验目标设计

### 4.1 从「想做什么」到「玩家感受到什么」

设计前必答：

1. 目标用户是谁？（巴图类型分布）
2. 核心 30 秒/5 分钟/1 小时体验是什么？
3. 玩家凭什么明天还来？

### 4.2 兴奋点（节奏）二维表

| 时间轴 | 兴奋点类型 | 强度(0-100) |
|--------|------------|-------------|
| 0-5min | 首次战斗/首充奖励 | 85 |
| 5-30min | 首次升级/新技能 | 70 |
| … | … | … |

**原则：**
- 前期兴奋点密集，后期间隔拉长但单点更强
- 松弛有度，《植物大战僵尸》是典范
- 疲劳型活动不宜过密

## 五、题材与玩法

| 维度 | 作用 |
|------|------|
| 题材 | 包装、用户筛选、美术/文案统一 |
| 玩法 | 核心操作与乐趣来源 |

**二者互为表里**，题材决定用户群体，玩法决定留存深度。

### 玩法两大训练类型

| 类型 | 特征 | 代表 |
|------|------|------|
| 行为训练 | 手眼反应、实时操作 | FPS、格斗、ACT |
| 思维训练 | 脑力策略、最优解 | 解谜、卡牌、棋类 |
