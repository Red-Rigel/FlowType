# 01 策划定位与分工

## 一、游戏策划的定义

### 1.1 是什么

游戏策划（Game Designer）负责：

- 设计**规则**与**内容**，定义玩家与游戏世界的交互方式
- 将创意转化为程序、美术可执行的**需求文档**
- 跟进需求落地，对**最终体验**负责
- 在研发全周期中充当**沟通枢纽**

> 用一句话概括：**构造虚拟游戏世界一切细节，使玩家在规则中获得精神满足。**

### 1.2 不是什么

| 误解 | 事实 |
|------|------|
| 就是天天玩游戏 | 玩游戏是调研手段，核心是设计与文档 |
| 只需要创意 | 创意 < 可执行性 < 体验验证 |
| 数学天才才能做数值 | 高中数学 + Excel + 体验直觉即可入门 |
| 写文档的工具人 | 文档是沟通媒介，价值在于推动正确体验落地 |

## 二、团队中的位置

```
         制作人/项目总监
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
  策划      程序      美术
 (需求方)  (实现方)  (表现方)
    │         │         │
    └─────────┴─────────┘
              │
           测试/运营
```

**腾讯语境下的策划角色：**

- 需求的发起者与跟进者
- 对最终结果负责的岗位
- 游戏制作人往往由策划出身

## 三、职能分工详解

### 3.1 系统策划

**职责：**
- 游戏功能模块设计（背包、好友、公会、商城…）
- 交互流程与 UI 逻辑
- 系统间关联与解锁节奏
- 活动策划与版本内容规划

**日常：**
- 撰写系统策划案 → 评审 → 拆需求 → 跟进 → 验收
- 维护配置表结构
- 与运营对接活动需求

**核心素质：** 逻辑能力 + 产品思维 + 沟通表达

**避坑：** 避免成为「文档工具人」—— 主动参与验收、推动执行、用图表降低沟通成本。

### 3.2 数值策划

**职责：**
- 战斗公式、属性成长、怪物数值
- 经济系统（产出/消耗/通胀控制）
- 商业化数值（付费点、礼包定价）
- 体验节奏（升级时间、战力曲线）

**核心公式：**
> 优秀数值策划 = 数学基本功 + Excel/VBA + **对体验的深刻理解**

详见 `04-数值策划方法论.md`

### 3.3 关卡策划

**职责：**
- 地图布局、动线设计
- 怪物/遭遇配置
- 难度曲线与教学关卡
- 关卡白盒搭建与迭代

**关卡的双重功能：**
1. **考验玩家** — 难度、技巧
2. **表达内容** — 剧情、氛围、系统教学

### 3.4 文案策划

**职责：**
- 世界观架构、剧情主线/支线
- 角色设定、对白、任务文本
- IP 延展与品牌一致性

详见 `06-叙事与世界观.md`

### 3.5 战斗策划

**职责：**
- 技能设计、战斗公式
- 动作节奏、打击感需求
- PVE/PVP 平衡
- 与程序协作战斗框架

### 3.6 主策划

**职责：**
- 把控游戏整体结构与体验一致性
- 各系统模块审核与取舍
- 版本规划与优先级决策
- 通常由上述四类策划晋升

## 四、行业分工趋势

| 趋势 | 说明 |
|------|------|
| 分工细化 | 越细岗位，专业深度要求越高 |
| 复合能力 | 小团队要求 T 型策划（一专多能） |
| 数据驱动 | 经分、用户研究成为标配技能 |
| 品类专精 | FPS/MOBA/SLG 等各有独立方法论 |

**校招常见方向：** 系统、文案、数值（关卡/战斗校招较少）

## 五、游戏内容三大板块

```
┌──────────────────────────────────────┐
│              游戏内容                 │
├──────────┬───────────┬───────────────┤
│  战斗/玩法 │  成长/经济 │  世界/叙事    │
│  (规则)   │  (目标)   │  (包装)       │
├──────────┼───────────┼───────────────┤
│ 系统策划  │ 数值策划  │ 文案策划      │
│ 战斗策划  │           │ 关卡策划      │
└──────────┴───────────┴───────────────┘
```

**美术方向**由美术负责人主导，其余板块基本一一对应策划工种。

## 六、入行能力模型

### 6.1 通用能力

| 能力 | 说明 |
|------|------|
| 逻辑思维 | 理清因果、状态、边界条件 |
| 文档能力 | 结构清晰、可执行、可检索 |
| 沟通协作 | 倾听、共情、精准表达 |
| 游戏阅历 | 深度体验 + 结构化拆解 |
| 工具使用 | Excel、思维导图、原型工具、引擎基础 |

### 6.2 作品集建议

1. **做一个小而完整的游戏/Mod** — 规模可控，体现全流程
2. **拆解 3-5 款竞品** — 附结构化分析，非流水账
3. **完成一份完整数值表** — 含战斗公式 + 成长曲线
4. **学习多种工具** — Unity/Unreal 基础、Figma、配置表

### 6.3 面试常考点

- 你最爱的游戏，**为什么好玩**？（体验层分析）
- 设计一个 XX 系统，你会怎么做？
- 数值题：伤害公式、掉落期望、成长曲线
- 与程序/美术意见不合如何处理？
