# 游戏策划方法论总览

## 一、方法论定位

本体系提炼自腾讯游戏学堂公开知识库 151 篇策划相关文章，核心思想与腾讯内网 `game.woa.com/knowledge/1013` 知识专栏一脉相承：**以体验为中心、以结构化为工具、以验证为闭环**。

### 1.1 游戏策划的本质

| 维度 | 定义 |
|------|------|
| **工作对象** | 特定规则 + 精神满足 |
| **核心产出** | 可执行的系统设计、数值模型、关卡内容、叙事表达 |
| **团队角色** | 需求发起者、体验负责人、跨职能沟通枢纽 |
| **成功标准** | 玩家持续投入（留存/付费/传播）+ 团队可落地执行 |

### 1.2 策划三分法（职能分工）

```
                    主策划（全局把控）
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
    系统策划          数值策划          关卡/战斗策划
    （功能模块）      （平衡与经济）     （内容与节奏）
        │                │                │
        └────────┬───────┴───────┬────────┘
                 ▼               ▼
             文案策划         技术策划
           （世界观/IP）    （工具/脚本）
```

| 岗位 | 核心职责 | 关键能力 |
|------|----------|----------|
| 系统策划 | 功能模块、交互流程、系统架构 | 逻辑 + 产品思维 + 表达 |
| 数值策划 | 战斗/经济/成长曲线 | Excel/VBA + 体验直觉 |
| 关卡策划 | 地图、动线、遭遇、难度曲线 | 空间感 + 节奏感 |
| 文案策划 | 世界观、剧情、角色、对白 | 文学功底 + 共情 |
| 战斗策划 | 技能、动作、打击感、平衡 | 品类专精 + 迭代能力 |

## 二、顶层设计框架（必读）

### 2.1 体验金字塔

```
        ┌─────────────┐
        │  商业目标    │  留存、付费、口碑、竞品差异化
        ├─────────────┤
        │  动机与节奏  │  成就、收集、社交、赌博感、兴奋点分布
        ├─────────────┤
        │  感受层      │  情绪、难度、数值体感、成就感
        ├─────────────┤
        │  表象层      │  美术、操作、界面、关卡、剧情
        └─────────────┘
```

### 2.2 三大分析框架

| 框架 | 用途 | 详见章节 |
|------|------|----------|
| **MDA** | 机制→动态→美学，从系统层分析游戏 | `02-核心设计框架.md` |
| **巴图四象限** | 玩家动机分类，指导内容配比 | `02-核心设计框架.md` |
| **五层分析法** | 从表象到商业目标的结构化拆解 | `08-竞品拆解与分析.md` |

### 2.3 设计双螺旋

```
    规则构建 ◄────────────────► 目标供给
    (核心玩法)                  (成长系统)
         │                           │
         ▼                           ▼
    学习→操控→反馈→成长        量变→质变→激励
         │                           │
         └───────────┬───────────────┘
                     ▼
              玩家持续投入
```

- **规则构建**：ACT 向内容，强调技巧、对抗、挑战（Killer/Achiever）
- **目标供给**：Interacting 向内容，强调收集、强化、探索（Socializer/Explorer）

现代商业游戏几乎都需要两者结合。

## 三、全流程工作法

### 3.1 策划生命周期

```mermaid
flowchart LR
    A[立项/定位] --> B[核心玩法验证]
    B --> C[系统设计]
    C --> D[数值建模]
    D --> E[内容生产]
    E --> F[测试迭代]
    F --> G[上线运营]
    G --> H[数据复盘]
    H --> B
```

### 3.2 各阶段核心动作

| 阶段 | 输入 | 输出 | 验证方式 |
|------|------|------|----------|
| 立项 | 市场、用户、竞品 | GDD 核心一页纸 | 纸面模型/原型 |
| 系统设计 | 核心体验目标 | 系统逻辑图 + 流程图 | 评审 + 白盒 |
| 数值设计 | 系统逻辑图 | Excel 数值表 + 体验目标 | 模拟 + 内测 |
| 关卡设计 | 数值标模 + 技能库 | 关卡白盒 + 遭遇表 | 可玩 Demo |
| 上线后 | 经分数据 + 反馈 | 优化方案 | A/B + 版本迭代 |

### 3.3 文档价值工程原则

> 前期多投入 1 小时做图解/白盒，可减少后期 10 小时跨职能沟通。

- **程序** → 逻辑流程图、状态机图
- **美术** → 示意图、参考图、尺寸规范图
- **策划内部** → 系统逻辑图、玩法维度图

## 五、十条黄金法则

1. **做数值就是做体验** — 公式服务于可感知的成长，而非数学优雅
2. **问题与解法成对出现** — 单独增加机制或挑战都是浪费
3. **知道做什么比怎么做更重要** — 需求定位优先于创意发散
4. **抽象文字不如具象图表** — 流程图、示意图是策划的基本功
5. **好游戏都是调出来的** — 预留迭代空间，拒绝一次定稿
6. **系统逻辑图先于数值表** — 理清模块影响再切数值
7. **时间是最硬的货币** — 所有数值用「获得时间」衡量价值
8. **反拆是学习最快的方式** — 逆推竞品比闭门造车有效
9. **玩法创造靠日积月累** — Dota/PUBG 皆从 MOD 改良而来
10. **策划要对最终结果负责** — 验收不只是 PM 的事
