# 05 关卡与战斗设计

## 一、关卡设计概述

### 1.1 关卡的双重使命

1. **考验玩家** — 技巧、策略、数值
2. **表达内容** — 剧情演绎、氛围营造、系统教学

> 精彩的游戏让玩家在关卡中**演故事**，而非只看过场动画。

### 1.2 关卡难度设计的真正难点

不是做出高难度，而是**适配玩家当前水平**：

- 挫折过强 → 流失
- 过易 → 成就感不足

**分析方法：**
- 记录每关通关时间
- 成就感/挫折感各打 1-10 分
- 绘制难度分布曲线

**标杆：** 《植物大战僵尸》— 难度递进 + 玩法穿插 + 松弛有度

## 二、PVE 关卡设计

### 2.1 手游 PVE 特性

| 特性 | 设计应对 |
|------|----------|
| 关卡时长短 | 模块化空间尺寸，组合复用 |
| 资源复用高 | 标准化空间类型 + 组合规则 |
| 线性流程为主 | 怪物组合 > 场景纵深 |
| 内容消耗快 | 高效白盒 + 图像化策划案 |

### 2.2 空间类型规划

根据角色尺寸、移速、镜头、UI 确定基准空间 → 推导遭遇空间类型：

```
小型遭遇区 / 中型战斗场 / Boss 竞技场 / 走廊过渡区
     ↓              ↓              ↓
  尺寸标注      怪物上限        动线示意
```

### 2.3 AAA 关卡模式策略

| 模式 | 说明 |
|------|------|
| 教学关 | 单一机制引入，低惩罚 |
| 复合关 | 多机制组合，考验掌握 |
| 压力关 | 资源限制、时间限制 |
| 奖励关 | 低难度高回报，情绪舒缓 |
| Boss 关 | 阶段机制，记忆点 |

### 2.4 资源有限时的重复挑战设计

- 随机要素（怪物组合、词缀）
- 评分/multi 目标（时间、无伤、收集）
- 难度自选档位
- 每周轮换 modifier

## 三、战斗设计

### 3.1 战斗节奏案例：《Apex 英雄》

**核心体验：** 爽快对战 + 丰富手段 + 开放地图战术

| 模块 | 设计点 | 对核心的贡献 |
|------|--------|--------------|
| 复活机制 | 倒地盾 + 复活道具 | 降低死亡代价，鼓励交战 |
| 功能性技能 | 探查/机动/救助/陷阱… | 丰富对战维度 |
| 关卡节奏 | 高遮挡分区、少迂回 | 控制交战频率与距离 |

**分析方法：** 先提炼 1 句核心体验 → 再拆模块论证点线面关系

### 3.2 MOBA 英雄平衡

**平衡维度：**
- 英雄间差异：属性、技能、伤害类型、攻击方式、移速
- 伤害公式中各属性权重与性价比
- 属性来源占比：角色/符文/装备
- 单局经济节奏：金币→属性的转化效率

**数学题本质：** 公平竞技下的职业平衡

### 3.3 FPS 设计要点

- 地图分区控制交战距离
- 武器差异化（射程/伤害/射速/弹道）
- 复活与节奏控制
- 音效/空间音频增强定位感

## 四、战斗公式与体验

### 4.1 常用公式

```
减法：伤害 = MAX(攻击 - 防御, 最低伤害)
乘法：伤害 = 攻击 × 技能系数 × (1 - 减伤率)
```

### 4.2 标模验证

设定标准战斗时长、耗血比例，反推怪物属性：

- 1v1 玩家耗血 80% 击杀 ← 常见 PVE 标模
- 团战 TTK（Time to Kill）← PVP 标模

### 4.3 多对多平衡

- 兰切斯特方程（兵力对比与战损）
- 有效生命 EHP × 有效输出 EDPS 框架
