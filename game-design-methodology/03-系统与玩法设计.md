# 03 系统与玩法设计

## 一、系统设计方法论

### 1.1 系统逻辑图（数值前置）

在数值设计之前，与相关策划共建**系统逻辑图**：

```
    [主线关卡] ──产出──► [金币] ──消耗──► [装备强化]
         │                    │
         ▼                    ▼
    [经验/等级] ◄── [每日任务]    [战力提升]
         │
         ▼
    [功能解锁] ──► [新系统入口]
```

**目的：**
- 理清系统间影响与联系
- 全盘考虑后再切分各模块数值
- 避免局部优化破坏全局平衡

### 1.2 系统设计六步法

| 步骤 | 动作 | 产出 |
|------|------|------|
| 1 | 定位需求核心点 | 需求一句话 + 3 个核心约束 |
| 2 | 预设问题，打草稿 | 设问清单 |
| 3 | 推导功能模块 | 模块列表 + 关联图 |
| 4 | 对口沟通（程序/美术） | 修订版策划案 |
| 5 | 图示化表达 | 流程图/示意图/白盒 |
| 6 | 评审 → 拆需求 → 跟进 | 可执行需求单 |

### 1.3 需求定位案例（PVP 战斗）

| 核心点 | 决策 |
|--------|------|
| 同步 vs 异步 | 同步 PVP（实时对抗） |
| 匹配方式 | 随时匹配真人 |
| 平衡性 | 相对平衡，允许适度付费优势 |

→ 推导出：战斗模块、关卡模块、匹配模块、AI 模块、数值天秤

## 二、文档表达规范

### 2.1 面向不同职能

| 对象 | 偏好 | 推荐形式 |
|------|------|----------|
| 程序 | 逻辑清晰、边界明确 | **流程图**、状态机、判定表 |
| 美术 | 直观、少文字 | **示意图**、参考图、尺寸标注 |
| 策划内部 | 可检索 | 目录 + 模块化章节 |
| 测试 | 用例可导出 | 条件-结果表 |

### 2.2 价值工程判断

**何时投入图解成本？**

- ✅ 跨职能、复杂流程、外包协作、高频沟通模块
- ❌ 一次性简单需求、团队已共识的常规内容

> 前期多 1 小时图解 ≈ 后期少 10 小时返工沟通

## 三、玩法创新路径

### 3.1 现实：很少从零创新

- Dota ← War3 MOD，经年改良
- PUBG ← DayZ MOD，经年改良
- **未来玩法创造：日积月累 > 平地惊雷**

### 3.2 三条创新路径

```
路径 A：机制组合    ── 新机制 + 旧机制重组
路径 B：动态改良    ── 反拆竞品 → 维度取舍 → 体验升级
路径 C：审美换皮    ── 高风险，需制作者审美能力极强
```

### 3.3 在反拆基础上构建新玩法

1. 用 MDA/玩法维度法完整反拆参考作
2. 明确要保留的体验核心（1-2 句）
3. 识别可替换的维度（节奏/空间/资源/信息…）
4. 纸面模型验证 → 白盒 → 小范围测试
5. 迭代调优

## 四、SLG 大型玩法设计案例（赤壁之战）

### 4.1 中后期困局诊断

| 问题 | 数据/现象 |
|------|-----------|
| 成长瓶颈 | 宫殿 20 级后留存爬升 |
| 生态固化 | 皇城战归属固定头部联盟 |
| 旧玩法失效 | 跨服国战战损高、指挥难、PVP 规避 |
| 参与度低 | 黄金联赛仅管理层操控 |

### 4.2 破局设计原则

1. **多服匹配** — 打破单服平衡
2. **军团规模** — 50v50，策略可施展
3. **低战损** — 激活跃 PVP 意愿
4. **势均力敌** — 参战人员战力匹配 + 胜负分

### 4.3 玩法结构

| 要素 | 设计 |
|------|------|
| 胜负判定 | 军团积分 |
| 个人追求 | 个人积分段位 → 奖励 |
| 空间博弈 | 建筑占领优先级（帝陵 > 箭塔 > 要塞） |
| 策略层 | 迁城次数、水晶分配、BUFF 选择 |
| 翻盘机制 | 龙灵/巨兽高额积分、箭塔封锁 |
| 节奏分段 | 0-5min 外围 → 10-30min 核心 → 30-60min 决战 |

### 4.4 可复用的 SLG 设计模式

```
诊断数据 → 定位瓶颈 → 拆解旧玩法痛点 → 设定体验目标
    → 匹配规则 → 积分双轨 → 空间物件 → 节奏分段 → 灰度验证
```

## 五、常见系统模块清单

| 模块 | 设计要点 |
|------|----------|
| 任务系统 | 引导节奏、奖励可视化、成就挂钩 |
| 商城/付费 | 价值感稳定、首充/月卡/礼包分层 |
| 社交/公会 | 责任绑定、协作玩法、排行榜 |
| 活动系统 | 时间窗口、稀缺感、回流激励 |
| 匹配系统 | ELO/战力区间、等待时间权衡 |
| 背包/装备 | 格子压力、强化路线、套装 |
