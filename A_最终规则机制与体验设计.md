# 方向 A ·《林记侨批局：寄给亡者的批》最终规则机制与体验设计（GameRes 优化版）

> 版本：v0.6 GameRes-informed optimization  
> 基于文件：`uploads/__A_____________cba4.md`  
> 参考资料：`output/gameres_game_design/` 下 91 篇 GameRes「游戏设计」专辑文章与《游戏设计方法论精粹.md》  
> 优化目标：不推翻原案，在 1-2 周 Web MVP 边界内，让本作更像“可玩的叙事解谜”，而不是“带 UI 的剧情文档”。

---

## 0. 优化结论先行

当前方案的方向是成立的：**单封主批、关系板、固定状态机、LLM 只做表达、不编真相**，都非常适合小体量叙事解谜。

但如果按 v0.5 直接制作，最大风险有 5 个：

| 风险 | 具体表现 | 对应 GameRes 方法论 | 优化方向 |
|---|---|---|---|
| 玩家仍觉得“只是在读剧情” | 圈疑点、查记录、问 NPC 可能变成点按钮拿文本 | 《Florence》：交互即情感；《完美的一天》：机制传达主题 | 把每个关键真相改成一个“动作谜题” |
| 记忆成本偏高 | 林素琴 / 阿琴 / 小满 / 陈永安 / 旧批 / 药方 / 退批封容易混 | 《艾迪芬奇》：减轻记忆成本、碎片边界相连 | 做“拼图边界提示”和“证据摘要句” |
| 地点只是菜单 | 药铺/祠堂/码头目前功能明确，但空间行为弱 | 《关卡设计集》：关卡塑造行为；《隐秘的角落》：流程与细节微妙 | 每个地点绑定一个独特小交互与情绪峰值 |
| NPC 像信息容器 | 证据到达后吐线索，行为连续性不足 | 《互动角色行为12原则》：行为可目击、连续、可中断、可解读 | 给 NPC 做“可观察行为状态”而不只改台词 |
| 结局像数值判定 | 承认/保护/规矩轴有设计，但玩家可能感不到“盖印重量” | 《Florence》：形状/操作变化表达情绪；峰终定律 | 终局拆成“摆事实-选语气-推印-犹豫”四段 |

## 1. 最终定位（保留，但压成更强的产品句）

### 一句话

> **1934 年，潮州府林记侨批局收到一封从暹罗寄来的侨批。收件人林素琴已故七年，玩家要用批脚、旧批、药方、族谱和证词，把这封银信真正应承认的人重新拼出来。**

### 玩家幻想

原案的幻想非常好，建议进一步游戏化：

> 我不是在听一个悲情故事，而是在亲手整理、比对、追问、盖印。  
> 每一条红线都是我接上的；最后那枚印，是我让制度对一个活人让步。

### 体验关键词

| 关键词 | 解释 | 设计抓手 |
|---|---|---|
| 纸上侦探 | 推理来自文书矛盾，不来自自由破案 | 批脚号、药方日期、族谱划痕、退批封 |
| 微开放 | 地点顺序可选，但真相链条有固定骨架 | 三地点任意调查 + 关系板门槛 |
| 温柔而沉重 | 不是恐怖，不是狗血，是“承认一个人”的重量 | 霉红封、印泥迟疑、孩子咳嗽、退批堆 |
| 低成本高密度 | 不做大地图，用物件、声音、UI 变化承载世界 | 物件状态、折叠摘要、NPC行为状态 |

| 《怎样让玩家畅玩文本量巨大的游戏》 | 阅读要有收益；文本短、位置好、可操作；物件文本比长段说明更易读 | 单段 NPC 回复 40-70 字；关键文本转成线索卡、票根、标记、摘要句 |
| 《互动角色行为12原则》 | 行为是可目击、连续、交互、可中断、可解读的 | NPC 状态不只变台词，还变动作：停算盘、藏药方、压族谱、避开眼神 |

## 3. 设计支柱（优化版）

### 支柱 1：一封信就是一张关系网

侨批不是剧情文本，而是可操作的关系证据。  
每个文书物件必须回答三件事：

1. **它证明了谁和谁有关？**
2. **它和另一份记录哪里矛盾？**
3. **它让玩家获得哪个新动作？**

如果某物件只能提供背景氛围，不能提供动作或关系，就降级为 P1 装饰。

### 支柱 2：任务从疑点长出来

保留原案“不做传统任务清单”。但要补一层：每个疑点要生成一种明确调查动作。

| 疑点类型 | 例子 | 生成动作 |
|---|---|---|
| 人名疑点 | 林素琴 / 阿琴 / 小满 | 查名册 / 族谱 / 药方 |
| 钱款疑点 | 附银叁拾 / 欠账 | 查账簿 / 药钱 |
| 时间疑点 | 已故七年 / 多年退批 | 排旧批时间线 |
| 关系疑点 | 孩子咳喘 / 银锁 | 连接人物和物件 |
| 规则疑点 | 是否能代收 / 是否能盖印 | 问爷爷 / 触发终局规则 |

### 支柱 3：文本必须变成可操作物

参考《怎样让玩家畅玩文本量巨大的游戏》：只要给玩家半个不读的理由，玩家就会跳过。

所以：

- 长文本不能直接堆在屏幕上；
- 每段文本要么能圈，要么能拖，要么能折叠成证据卡；
- NPC 不能连续输出超过 2 句信息密集台词；
- 所有关键信息都要有“非文本备份”：颜色、票根、划痕、声音、动作。

### 支柱 4：NPC 是“会被证据改变行为的人”

NPC 不只是知识边界，而是行为边界。

| NPC | 初始可目击行为 | 证据后行为变化 | 玩家读到的意义 |
|---|---|---|---|
| 爷爷 | 擦印章、拨算盘，不看玩家 | 看到旧批后停算盘，盖上旧柜 | 他知道旧事，但被规矩压住 |
| 阿娴 | 把药方簿反扣 | 看到附银后把药方推近半寸 | 她想救孩子，但怕祠堂 |
| 王婆 | 手压族谱划名页 | 看到银锁后合上族谱 | 她维护的是体面，不是真相 |
| 阿水 | 把退批封当废纸捆 | 对上批脚号后拆开绳结 | 他知道这不是第一次 |

### 支柱 5：结局不是选择，而是一次仪式

玩家不是点“认名/留面/退批”。玩家做四个动作：

```text
摆事实卡 → 选语气卡 → 推印章 → 看爷爷是否放手
```

最终结局来自这些动作共同表达的价值取向。

## 4. 核心循环（优化版）

### 4.1 0-30 秒微循环

```text
看见异常
→ 圈出疑点
→ 疑点变成待核卡
→ 去对应地点执行一个小动作
→ 得到可拖线索卡
→ 钉到关系板
→ 关系板反馈“新边界”
→ NPC 出现新行为 / 新追问
```

### 4.2 40 分钟试玩循环

```text
普通来批：学会银信合一
→ 异常来批：寄给亡者
→ 三地点任选：药铺 / 祠堂 / 码头
→ 关系板第一次闭环：陈永安-林素琴-小满
→ 旧批柜打开：时间线确认
→ 爷爷评判：规则压力
→ 回批落印：价值表达
→ 后日谈：世界继续营业
```

### 4.3 “总-分-总”结构

参考《完美的一天》：

| 段落 | 功能 | 本案对应 |
|---|---|---|
| 总 | 给出核心谜面 | 主批：死人收批 |
| 分 | 玩家自由拼碎片 | 药铺/祠堂/码头/旧批 |
| 总 | 回到核心谜面重估意义 | 回批：这封批该承认谁 |

## 5. MVP 内容边界（收束优化）

原案 P0 内容略多，建议压成“6 个必做 + 4 个延后”。

### 5.1 P0 必做

| 模块 | 数量 | 说明 |
|---|---:|---|
| 普通来批 | 1 封 | 只教“银信合一”和圈疑点，不做 2 封 |
| 异常主批 | 1 封 | 核心谜面 |
| 旧批 | 2 封 | 一封确认阿琴，一封确认银锁/连续寄钱 |
| 地点 | 4 个 | 侨批局、药铺、祠堂、码头 |
| 核心 NPC | 4 个 | 爷爷、阿娴、王婆、阿水 |
| 关键线索物 | 6 个 | 药方、旧批、银锁、族谱划痕、退批封、批脚号 |
| 关系节点 | 7 个 | 陈永安、林素琴、阿琴、小满、爷爷、王婆、阿水 |
| 耦合公式 | 8 条 | 保持原案 C01-C08 |
| 结局 | 3+1 | 认名、留面、退批、完整回批 |
| LLM | 4 类 | NPC、提示、爷爷评判、回批润色 |

### 5.2 P1 延后

| 模块 | 延后理由 |
|---|---|
| 第二封普通来批 | 教学冗余，易拖慢进入主谜面 |
| 背景 NPC | 世界温度可先靠声音和物件实现 |
| 更多后日谈卡 | 先保证一个结局有力 |
| 更多地点细节 | P0 只做“每地点一个强交互” |

### 5.3 1-2 周 Web MVP 最小体验目标

> 玩家在 40 分钟内，至少经历 5 次“我自己发现了”的时刻。

| 时刻 | 触发 |
|---|---|
| 1 | 圈出“林素琴”后发现名册显示已故 |
| 2 | 药方查到“小满咳喘” |
| 3 | 旧批里“阿琴”与林素琴对上 |
| 4 | 批脚号对上多年退批 |
| 5 | 关系板连出小满真实位置 |

## 6. 四个地点的“微关卡”设计

参考《关卡设计集》和《关卡设计六要素》，每个地点都不是资料库，而是一段微型关卡。

### 6.1 侨批局：规则起点

| 要素 | 设计 |
|---|---|
| 起因 | 普通来批教学后，异常主批出现 |
| 玩家动作 | 拆批、圈疑点、查名册、开关系板 |
| 空间行为 | 从柜台到旧批柜；爷爷在印章旁形成压力 |
| 峰值 | 名册显示“林素琴：殁，民国十六年” |
| 新机制 | 圈疑点、待核卡、关系板空节点 |

### 6.2 药铺：让小满从文字变成活人

| 要素 | 设计 |
|---|---|
| 起因 | 主批里“孩子咳喘” |
| 玩家动作 | 筛药方簿：小孩 / 咳喘 / 欠账 |
| 空间行为 | 阿娴先挡住药方簿，证据后让出半页 |
| 峰值 | 背景一声孩子咳嗽，药方边缘有小指印 |
| 新机制 | 线索卡“药方_小满”；解锁小满节点 |

### 6.3 祠堂：家族秩序的阻力

| 要素 | 设计 |
|---|---|
| 起因 | 主批收件人已故，但寄信人仍寄 |
| 玩家动作 | 翻族谱、对年份、观察划痕 |
| 空间行为 | 王婆手压族谱；玩家必须先问后看 |
| 峰值 | “林素琴”一行被新墨覆盖旧墨 |
| 新机制 | 线索卡“族谱划痕”；规矩轴压力出现 |

### 6.4 码头：海外联系仍在继续

| 要素 | 设计 |
|---|---|
| 起因 | 批脚号需要查船期/退批 |
| 玩家动作 | 对批脚号、查退批封、排年份 |
| 空间行为 | 阿水把退批封当废纸捆，玩家证据足才拆 |
| 峰值 | 多封退批封批脚连续，说明不是误寄 |
| 新机制 | “多年退批”事实卡；解锁陈永安连续来批 |

## 7. 信息拼图骨架

### 7.1 线索不是散点，而是拼图边界

每张线索卡新增 3 个字段：

```json
{
  "card_id": "clue_prescription_xiaoman",
  "proves": ["小满存在", "小满咳喘"],
  "connects_to_types": ["主批疑点:孩子咳喘", "事实卡:钱可救急", "人物:阿娴"],
  "summary": "药方簿证明小满长期咳喘，且欠过药钱。"
}
```

### 7.2 关系板反馈规则

每次成功连接，给三层反馈：

| 层级 | 表现 | 目的 |
|---|---|---|
| 视觉 | 红线收紧、节点由问号变半透明头像 | 告诉玩家“接上了” |
| 文本 | 生成一句证据摘要 | 减轻记忆成本 |
| 行为 | 某 NPC 新追问/新动作 | 让世界响应玩家 |

### 7.3 证据摘要句

参考《艾迪芬奇》的“减轻记忆成本”，关系板上每条确认关系都要能一句话复述。

| 关系 | 摘要句 |
|---|---|
| 陈永安 → 林素琴 | “旧批证明陈永安多年称林素琴为阿琴。” |
| 林素琴 → 小满 | “银锁与药方共同指向小满是阿琴留下的孩子。” |
| 陈永安 → 小满 | “这笔钱表面寄给亡者，实际能救仍活着的小满。” |
| 林记 → 退批 | “批脚记录显示，林记多年按规矩退回过这类来批。” |

## 9. NPC 行为状态机

### 9.1 NPC 状态不只改文本

| NPC | State 0 初见 | State 1 第一线索 | State 2 冲突证据 | State 3 终局前 |
|---|---|---|---|---|
| 爷爷 | 擦印章；说按名册办 | 停算盘一下 | 把旧柜钥匙压在掌心 | 印章推到玩家够得到处 |
| 阿娴 | 药方簿反扣 | 露出半页药方 | 把“小满”名字划给玩家看 | 说“别让孩子再被赶一次” |
| 王婆 | 手压族谱 | 翻页变慢 | 合上族谱，挡住划名 | 站到祠堂门口形成阻力 |
| 阿水 | 整理船单 | 认出批脚号 | 拆退批封绳结 | 提醒“船今晚走” |

### 9.2 NPC 回复长度规则

| 类型 | 字数 | 目标 |
|---|---:|---|
| 初见寒暄 | 20-40 | 建立人设 |
| 证据追问 | 40-70 | 给反应，不讲完 |
| 冲突证据 | 60-90 | 露出情绪和新事实 |
| 终局前 | 70-120 | 承接价值压力 |

### 9.3 LLM Prompt 增补：行为前缀

LLM 输出建议结构：

```json

  "observable_action": "阿娴把药方簿往回收了半寸，又停住。",
```

## 10. 文本减负与 UI 规则

### 10.1 文本呈现三层

| 层级 | 形式 | 字数 | 使用场景 |
|---|---|---:|---|
| 可读原文 | 信纸、药方、族谱原页 | 不限，但分块 | 玩家主动查看 |
| 证据摘要 | 线索卡一句话 | 20-40 | 关系板与回批选择 |
| 弱提示 | 爷爷/系统一句话 | 20-50 | 卡关、拖错 |

### 10.2 大段文字拆法

不直接展示：

> “陈永安多年寄钱给林素琴，但林素琴已故，林家因体面拒收，导致小满长期得不到承认……”

拆成 4 个可操作物：

| 信息 | 物件 |
|---|---|
| 多年寄钱 | 连续批脚号 |
| 林素琴已故 | 名册 / 族谱 |
| 因体面拒收 | 王婆状态 + 划名 |
| 小满未被承认 | 药方无姓 + 银锁 |

### 10.3 已核折叠规则

保留原案，并补充“重复查看必须有收益或直接跳过”。

| 重复行为 | 默认处理 |
|---|---|
| 再看主批 | 直接显示已圈疑点 + 未解决疑点 |
| 再看药方 | 显示“小满咳喘 / 欠账 / 日期”三标签 |
| 再问旧问题 | 折叠旧对话，只显示“追问新证据” |
| 再拖错同类线索 | 第二次后高亮可能目标类型 |

## 11. 终局回批与落印（重点优化）

### 11.1 终局四段式

```text
第一段：摆事实
第二段：选语气
第三段：爷爷评判
第四段：推印落印
```

### 11.2 事实卡分层

| 层级 | 示例 | 作用 |
|---|---|---|
| 必要事实 | 林素琴已故；小满存在 | 没有则不能完整回批 |
| 关系事实 | 阿琴=林素琴；银锁指向小满 | 决定承认轴 |
| 现实事实 | 药钱急迫；多年退批 | 决定保护轴 |
| 制度事实 | 林记可代管；爷爷信任 | 决定规矩轴 |

### 11.3 语气卡从“选项”改成“回批姿态”

| 语气卡 | 文本倾向 | 交互表现 | 代价 |
|---|---|---|---|
| 说真话 | 明确小满身份 | 字迹稳定，印章更难推 | 祠堂压力高 |
| 留体面 | 避免公开名字 | 字句变委婉，红线半透明 | 小满仍未被完全承认 |
| 促其回乡 | 给未来留口 | 结尾留船期 | 当下处理不彻底 |
| 退回规矩 | 按名册拒收 | 印章容易盖，但声音发空 | 玩家感到制度胜利、人失败 |

### 11.4 印章交互

参考《Florence》的交互情感化：

| 条件 | 印章手感 |
|---|---|
| 证据不足 | 印章拖到一半被爷爷按住 |
| 规矩高 | 印章很顺，但盖下后颜色淡 |
| 承认高 | 印章有阻力，需二次确认 |
| 完整回批 | 玩家推到最后，爷爷松手，印泥声音清晰 |

### 11.5 结局回声

| 结局 | 后日谈物件 | 一句话回声 |
|---|---|---|
| 认名 | 药方上第一次写全“小满” | “名字写上去，比银元还沉。” |
| 留面 | 药钱到账，名字仍空 | “病能缓，名还悬着。” |
| 退批 | 码头多一封退批 | “账本干净，信又回海上。” |
| 完整回批 | 旧柜钥匙留给玩家 | “林记的印，不盖给纸，盖给人。” |

## 12. 情绪曲线（优化版）

### 12.1 目标曲线

```text
安稳：普通来批，规则可依
→ 异常：死人收批
→ 不安：记录对不上
→ 参与：我接上第一条红线
→ 沉重：退批不是一次，是多年
→ 压力：承认会伤人，不承认也伤人
→ 仪式：推印、落印、回声
```

### 12.2 峰值设计

| 峰值 | 位置 | 设计方式 |
|---|---|---|
| 第一次心里一沉 | 名册显示已故 | 文书事实瞬间推翻主批表面 |
| 第一次主动发现 | 药方连到“孩子咳喘” | 玩家亲手连线，不由 NPC 告诉 |
| 中段反转 | 多年退批 | 从“误寄”变“关系被延迟” |
| 终局压力 | 爷爷按住印章 | 制度具象化成手的阻力 |
| 结束回声 | 后日谈物件 | 峰终定律，留下最后感受 |

## 13. Onboarding 优化

### 13.1 开场 5 分钟必须完成

| 时间 | 内容 | 玩家学到 |
|---:|---|---|
| 0:00-0:30 | 普通来批到桌上 | 这是侨批局 |
| 0:30-1:30 | 拆批，圈“银元/母亲” | 圈疑点 |
| 1:30-2:30 | 名册确认收件人 | 查记录 |
| 2:30-3:30 | 拖到关系板，盖收讫 | 线索→关系→结果 |
| 3:30-5:00 | 异常主批出现 | 规则被破坏 |

### 13.2 第一轮教学原则

- 第一封普通来批只教一个循环，不教药铺/祠堂/码头；
- 第一封必须成功，不能失败；
- 爷爷提示：“先看名，再看钱，再看话。”作为全局基础口诀；
- 异常主批必须在第 5 分钟前出现，否则核心卖点太晚。

## 14. Playtest 观测指标

### 14.1 必测问题

| 问题 | 通过标准 |
|---|---|
| 玩家是否知道下一步去哪？ | 80% 玩家不需强提示进入第一个地点 |
| 玩家是否能复述核心关系？ | 试玩后能说出“陈永安-林素琴-小满” |
| 玩家是否觉得自己在推理？ | 70% 以上提到“我发现/我连出来” |
| 玩家是否读得烦？ | 单段文本跳过率低；无连续 2 次关闭文本 |
| 玩家是否在乎小满？ | 试玩后主动提到孩子/药钱/名字 |
| 玩家是否理解结局不是好坏？ | 能说出自己结局的代价 |

### 14.2 卡点记录表

| 卡点 | 可能原因 | 修正 |
|---|---|---|
| 不会圈疑点 | 首次可交互提示不足 | 普通来批加动效框 |
| 不知道地点可去 | 任务从疑点生成不明显 | 待核卡上显示“可查：药铺” |
| 拖线索乱试 | 线索边界不清 | 卡面加“可能关联：人名/药方/批脚” |
| 忘记阿琴是谁 | 摘要不足 | 关系板固定显示别名 |
| 不在乎终局 | 小满存在感弱 | 药铺咳嗽声/小指印提前 |

## 16. 最终机制规格（优化后摘要）

### Mechanic 1：拆批与圈疑点

- **Purpose**：把信从文本变成问题生成器。
- **Success**：玩家 3 分钟内知道“圈疑点会生成调查方向”。
- **优化点**：每个疑点卡显示“可查地点/可关联类型”，避免玩家不知道下一步。

### Mechanic 2：关系板耦合

- **Purpose**：让玩家亲手拼出关系，而不是听 NPC 说真相。
- **Success**：玩家至少 5 次产生“这是我发现的”。
- **优化点**：每次成功连接都生成证据摘要句，降低记忆成本。

### Mechanic 3：证据追问 NPC

- **Purpose**：让 NPC 因证据改变行为。
- **Success**：玩家能感到“拿不同证据问同一个人，反应不同”。
- **优化点**：NPC 输出由“行为前缀 + 短台词 + 是否给新事实”组成。

### Mechanic 4：已核与折叠

- **Purpose**：避免解谜重复疲劳。
- **Success**：重复查看已核对象 3 秒内回到重点。
- **优化点**：已核不是纯折叠，而是显示“摘要 + 新高亮 + 可用动作”。

### Mechanic 5：回批与落印

- **Purpose**：把真相处理变成价值表达。
- **Success**：玩家理解“真相不是答案，怎么承认才是选择”。
- **优化点**：印章有阻力、迟疑、松手、淡印等状态，承载情绪。

## 17. 最终 30 秒 Demo 讲法（优化版）

> 《林记侨批局：寄给亡者的批》是一款 Web 叙事解谜。玩家在 1934 年的潮州侨批局收到一封从暹罗寄来的银信，但收件人林素琴已经死了七年。你要拆批、圈疑点、查药方、翻族谱、对批脚号，把旧批、退批封和 NPC 证词钉到关系板上，最终决定这封批该不该承认一个被家族抹去的孩子。LLM 不负责编真相，只在固定状态机下扮演 NPC；真正的推理来自玩家亲手接上的每一条红线和最后推下去的那枚印。
