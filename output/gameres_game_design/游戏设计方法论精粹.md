## 二、立项与产品方向

### 2.1 立项三维思维模型

**框架 → 体验 → 场景**（《立项可行性分析指南》）

| 维度 | 问自己什么 | 产出物 |
|------|------------|--------|
| **框架** | 核心循环是什么？数值成长还是策略构筑？ | 玩法框架图、核心循环表 |
| **体验** | 玩家30秒/30分钟/30小时分别感受什么？ | 体验曲线、峰值时刻列表 |
| **场景** | 谁在什么设备/时长/社交环境下玩？ | 用户场景画像、竞品错位点 |

### 2.2 机会点四类

1. **创新**——新机制/新题材，风险最高，需小步验证  
2. **完善**——成熟品类做体验升级（难，赛道拥挤）  
3. **细分**——重度体验中度化、重度品类体验深挖  
4. **题材**——换皮不够，需同步解决代入感与需求创造

### 2.3 轻/中/重度未来侧重（结论摘录）

| 类型 | 未来侧重 | 逻辑 |
|------|----------|------|
| **轻度** | 流量平台化 | 创意获量成本高，平台可规避连续成功率问题 |
| **中度** | 方向研究+融合 | 在获量或回收保一前提下错位竞争 |
| **重度** | 细分体验+成本竞争 | 用户流转变慢，需极致质量或场景创新 |

### 2.4 融合类产品立项检查

- [ ] 融合解决了哪个**未被满足的需求**？（不是"两个热门拼一起"）
- [ ] 融合带来的**设计冲突**是否逐一有解法？（节奏、数值、UI、学习成本）
- [ ] 团队能力是否匹配**最长板**而非"什么热做什么"？
- [ ] 2年后市场需求是否仍成立？（避免刻舟求剑）

### 2.5 案例：策略构筑 × 融合方向

《立项可行性分析指南》以"策略构筑"为例：卡牌/rogue构筑的核心是**每局决策密度**与**局外成长节奏**的平衡。融合时需明确——构筑是主菜还是调味？

## 三、关卡设计（Level Design）

### 3.1 定义与边界

> **关卡 = 游戏发生的空间**，设计师关注"空间如何塑造玩家行为"。（《关卡设计集》编译自 Level Design Book）

| 概念 | 狭义 LD | 广义 Level Design |
|------|---------|-------------------|
| 关注点 | 布局、遭遇战、引导、平衡 | 含场景美术、灯光、叙事环境 |
| 玩家感知 | 无区别——玩家体验的是整体 | 无区别 |

**房间设计 vs 世界设计**：
- 脚本化体验 → 像建筑师一样抠房间
- 系统/玩家驱动（BR、开放世界）→ 像城市规划一样做群落与流线

### 3.2 标准生产管线（6阶段）

```
前期制作 → 平面布局(Layout) → 体块测试(Greybox) → 脚本 → 场景美术 → 发布
```

| 阶段 | 核心产出 | 常见误区 |
|------|----------|----------|
| 前期制作 | 目标、节奏、验证点、情绪主调 | 跳过计划直接做图 |
| 平面布局 | 2D俯视/透视布局图 | 不画图指望别人读懂心思 |
| 体块测试 | 可玩灰盒、耗时测算 | 叙事关过度体块迭代 |
| 脚本 | 事件、AI、门、任务 | — |
| 场景美术 | 道具、灯光、材质 | **过早美术=返工地狱** |

### 3.3 关卡「故事六要素」（可复用模板）

《关卡设计六要素》将关卡视为微型故事：

| 要素 | 设计要点 |
|------|----------|
| **时间** | 晨昏雨雪影响氛围与玩法（视野、移动） |
| **地点** | 边界清晰、支路有回报、地标定位、场景叙事 |
| **人物** | 中立NPC（氛围/支线）vs 敌对NPC（验证/推进剧情） |
| **起因** | 进本前让玩家知道"为何而来、要做什么" |
| **经过** | 分阶段小目标 + 面包屑引导 |
| **结果** | 与主线衔接，可抛悬念 |

**面包屑引导工具箱**：跟随NPC、敌人分布、镜头、贴花、照明、运动物体、标识箭头

### 3.4 节奏（Pacing）与情感曲线

- **峰值体验**决定记忆——峰终定律（高峰+结束时的感受）
- 峰值需**错落分布**，避免刺激叠加导致麻木
- 关卡间用**玩法/场景/战斗方式差异**制造峰值（如《SIFU》夜店vs美术馆的明暗动静对比）

### 3.5 平面布局（Layout）关键概念

| 概念 | 作用 |
|------|------|
| **关键路径(Critical Path)** | 主线必经路线，控制叙事与难度递进 |
| **遭遇战(Encounter)** | 敌人配置+空间+资源的组合谜题 |
| **掩体(Cover)** | 射击关卡节奏控制器——好掩体=有取舍的攻防博弈 |
| **流线/导引** | 玩家无意识走向设计师意图 |

### 3.6 关卡设计方法论速查

```
1. 先问：这个空间要让玩家做什么行为？
2. 画布局：玩家能去哪、想去哪、被迫去哪
3. 灰盒验证：大小、困惑度、耗时、平衡
4. 再加美术：美术服务引导，不先于功能
5. 测试：看玩家实际路线 vs 设计路线
```

### 3.7 案例启示

| 游戏 | 可复用经验 |
|------|------------|
| **《Stray》** | 猫咪视角限制高度→天然引导；环境叙事与美术一体 |
| **《SIFU》** | 巷战切换侧视镜头→解决夹角视野问题，强化压迫感 |
| **《双截龙II》** | 地形杀=机制×地形的设计范式 |
| **《落叶城》** | 解谜关卡：机制教学→组合→峰值谜题的节奏 |

## 四、战斗与数值设计

### 4.1 数值策划元认知

> **数值不是万能的，是调节体验的工具和显示器。先有需求，再有数值。**（王鑫然，《战斗数值框架》）

| 角色 | 职责边界 |
|------|----------|
| 战斗策划 | 从"好玩"出发设计机制，大型项目不负责数值 |
| 数值策划 | 框架搭建、平衡计算、商业化测算 |
| 全体策划 | 都应理解数值，避免模块割裂成"缝合怪" |

### 4.2 基础概念工具箱

| 概念 | 定义 | 用途 |
|------|------|------|
| **回合** | 一次攻击结算的最小单位 | 平衡基准 |
| **数学期望** | Σ(概率×值) | 衡量波动数据的"真实水平" |
| **体验设定数据** | 自变量（战斗时长、击杀刀数…） | 数值计算起点 |
| **最大值/最小值** | 技能收益上下界 | 控制战斗容错高低 |
| **TTK/STK** | Time/Shot To Kill | 射击/动作游戏节奏核心 |

### 4.3 伤害公式选型：减法 vs 乘法

《数值方法论01·减法与乘法公式》核心对比：

| 维度 | 减法公式 `伤害=攻击-防御` | 乘法公式 `伤害=攻击×减伤率` |
|------|---------------------------|------------------------------|
| 代表 | 梦幻西游、三国志战略版 | LOL、阴阳师 |
| 本质 | 防御像"格挡值"，与攻击对抗 | 减伤率像"额外血量"，线性叠生存 |
| 优势 | 攻防接近时博弈激烈；数值可克制膨胀 | 后期收敛可控；多段低伤技能自由 |
| 劣势 | 攻击≈防御时伤害跳变；高频低伤难设计 | 不同等级难保持"皮厚"体验一致 |
| **实践** | 梦幻用修炼等做复合公式 | 拉格朗日：格挡+百分比减伤，均设90%上限 |

**战斗模型最简式**：`战斗回合数 = EHP / EDPS`

### 4.4 数值框架搭建四步

1. **定义属性**——生命、攻击、防御、暴击、命中…  
2. **定义公式**——伤害、减免、命中、暴击（系数可调）  
3. **定义判定流程**——命中→暴击→伤害→CD计算  
4. **定义攻击流程**——前摇/命中/后摇与CD起算点

**CD起算点影响技能DPS**：
- 前摇开始计CD → 同时间内释放次数最多
- 后摇结束计CD → 释放次数最少

### 4.5 平衡计算工作流

```
框架搭建（手填自变量）→ 大规模调整（按预期因变量反推自变量）→ 上线数据调优（永远算不准）
```

**关键原则**：玩家会用你预料不到的方式打破预期——数值是活的。

### 4.6 打击感设计框架

《打击感从何而来》（DNF/火影手游）：

| 模块 | 关键要素 |
|------|----------|
| **攻击动作** | 前摇蓄力、发力关键帧、节奏分配、中间过渡帧 |
| **受击动作** | 形变/位移、多类型受击、阻尼震动 |
| **判定框** | 受击框在身体内、攻击框覆盖认知范围、位置稳定 |
| **顿帧** | 命中时双方短暂定格，模拟阻力 |
| **多媒体** | 伤害数字、特效、音效、镜头/场景震动 |

**打击感本质**：还原玩家认知中的"打中了"——不是数值加减。

### 4.7 射击游戏战斗节奏：TTK光谱

| 游戏 | TTK倾向 | 体验 |
|------|---------|------|
| **彩虹六号** | 极短（~0.3s） | 一见面定胜负，紧张、信息战 |
| **APEX英雄** | 较长（~1s+） | 拉扯、跟枪、身法、持续交火 |
| **光环** | 中等 | 掩体peek、团队配合 |

**APEX设计原则**：
- 技能**辅助战斗**，不直接秒杀——核心是射击
- 高机动（滑铲/攀爬）→ 增遭遇率、拉长线TTK、强调跟枪
- 大体型角色用**减伤被动**而非加血量平衡

**彩六设计原则**：
- 防守方信息优势 → 进攻方枪械伤害更高
- 距离衰减限制防守方外出对枪
- 后坐力从随机改为固定→服务电竞化

### 4.8 卡牌/放置战斗文档要点

《从零开始·放置回合卡牌战斗篇》：
- 先定**战斗时长目标**和**养成节奏**
- 技能分类：主动/被动/触发，明确优先级
- 文档需含：属性表、公式、技能描述、边界case

## 五、叙事与角色设计

### 5.1 叙事结构选型

《非线性游戏叙事：收集式剧情设计》

| 类型 | 特征 | 代表 | 成本 |
|------|------|------|------|
| **线性** | 三幕结构，固定顺序 | 《神秘海域》 | 低 |
| **点状** | 剧情散落开放世界 | 各类开放世界 | 中 |
| **分支** | 选择导向不同节点 | 《底特律：变人》 | 高 |
| **平行** | 多视角同时发生 | 《生化危机2》双主角 | 高 |
| **拼图/收集式** | 碎片独立但互相关联 | 《艾迪芬奇的记忆》 | 高 |

**非线性叙事难点**：分支越多，资产量指数增长；需专用叙事工具（Word/Final Draft不够用）。

### 5.2 收集式叙事设计方法

以《艾迪芬奇的记忆》为例：

1. **收集载体**——族谱、笔记本、房间物品  
2. **环境叙事**——每个房间物品塑造人物（Calvin房间→宇航员梦想）  
3. **玩法即叙事**——每个家族成员死亡场景=独特玩法机制  
4. **拼凑时刻**——碎片集齐后的"豁然开朗"是情感峰值

### 5.3 共情设计（《最后生还者》）

游戏作为叙事媒介的优势：
- **互动=投入**——玩家亲手做出的选择权重更高
- **玩法与主题同构**——生存压力强化父女/同伴羁绊
- **节奏交替**——紧张战斗与安静探索交替，给情感呼吸空间

### 5.4 角色吸引力模型

《游戏人物的全景》

```
吸引力 = 价值(V) × f(可得性)
```

| 概念 | 说明 | 设计启示 |
|------|------|----------|
| **价值** | 广义：任何对受众有好处的东西（非仅钱/武力） | 明确角色提供什么价值 |
| **S价值** | 生存价值（男性魅力常源） | 能力、领导、智慧、勇敢 |
| **R价值** | 繁衍价值（女性魅力常源） | 生育信号、温柔、关怀 |
| **可得性** | 越容易得到，吸引力越低 | 角色不宜"倒贴"；死亡/离开=可得性归零→情感峰值 |

**反模式**：无铺垫的"工具人之死"只靠蒙太奇强拉情绪。

### 5.5 互动角色行为12原则（摘录）

- 行为需有**内在动机**和**可预测性**
- NPC应**回应玩家**而非各说各话
- 情绪变化需有**因果链**
- 避免"等待玩家触发"的僵硬感

### 5.6 大文本量游戏可读性

- 分段、变体字体、插图打破文字墙
- 关键信息做**玩法奖励**而非纯阅读
- 允许跳过但不允许看不懂核心动机

## 六、系统设计与品类专题

### 6.1 模拟经营核心公式

《三万字·模拟经营感悟》

```
发展（从小到大）+ 解谜（配置资源解题）+ 掌控（剥离现实的负面/干扰/不可控）
```

| 体验层 | 机制层 |
|--------|--------|
| 欣欣向荣的成长感 | 经营主体扩张、正向反馈累积 |
| 解题成就感 | 系统出题+玩家自定目标 |
| 秩序掌控感 | 剥离"受伤减益""不可预估冷箭""强制中断" |

**难题设计两要点**：
1. 难度恰到好处——需配置资源后才能解  
2. 破解反馈与难度匹配——难越高，奖励越大

### 6.2 养成系统六阶段心理

《为何手游都带养成》

```
初入惊艳 → 挑战受阻 → 成长愉悦(斯金纳) → 边际递减 → 新途径(装备/经脉) → 目标链固化 → 社交比较 → 生命周期延长
```

**策划目的**：玩家在养成投入时间/金钱，换取实力范围内的内容体验。

**风险**：坑太深、成长迟滞 → 流失。

### 6.3 生存游戏：从「生」到「存」

《重塑生存游戏认知》

| 要点 | 说明 |
|------|------|
| ≠战术竞技 | 大逃杀目标是"唯一存活"，生存游戏目标是"持续活下去" |
| 四维生存 | 食物、身体、精神、生命——行为有反馈 |
| 玩法不唯一 | 可建筑/驯养/钓鱼，非纯对抗 |
| 突破方向 | 从"活着"升级到"活得好"（生活质量、社交、创造） |

## 七、商业化与长线运营

### 7.1 商业化六问框架

《手游商业化设计体系》（弓长又欠）

| # | 问题 | 核心逻辑 |
|---|------|----------|
| 1 | 怎么激发需求？ | 无需求则无付费 |
| 2 | 怎么规划商品与价格？ | 分层匹配大R/中R/小R |
| 3 | 怎么展示商品？ | 锚定、稀缺、对比 |
| 4 | 怎么引导比较？ | 让玩家自己得出"划算"结论 |
| 5 | 怎么促成购买？ | 降低摩擦、限时压力 |
| 6 | 怎么提升购后满意？ | 避免付费即巅峰 |

### 7.2 需求强度定价法

**同一商品"体力"，玩家愿付价格随场景变化**：

| 场景 | 需求强度 | 心理价位 |
|------|----------|----------|
| 无目标中场休息 | 低 | 不买 |
| 活动即将结束 | 中 | ¥2 |
| 连败6次快通关 | 高 | ¥5 |
| 团队赛最后阶段 | 极高 | ¥8 |

**公式**：商品价值 ≈ 未满足痛苦的强度

### 7.3 需求三类

| 类型 | 特征 | 设计要点 |
|------|------|----------|
| **理性需求** | 进度受阻、资源耗光、收益降低 | 付费率较高，易逼氪 |
| **感性需求** | 炫耀、收集、情感 | 外观、限定、剧情 |
| **社交需求** | 团队压力、认同 | 公会活动、协作竞争 |

### 7.4 理性需求设计三板斧

```
1. 培养认知惯性（高胜率、无限体力、随意拾取…）
2. 用障碍阻断惯性（难关、体力空、背包满）
3. 玩家痛苦→消除痛苦的需求→商品出现
```

**关键换算**：让玩家清晰感知 **游戏道具 = 现实时间 = 现实金钱**

> 花¥6买道具难；花¥6省2小时无聊时光——容易接受。

### 7.5 付费分层测算示例

《战斗数值框架》摘录：

```
年流水目标 + 流量 + 付费率 + 大中小R占比
→ 反推单用户年ARPU → 决定付费深度与礼包定价
```

例：大R年均¥3200、中R ¥200、小R ¥12.5 → 对应首充/月卡/礼包档位设计。

## 八、社交与多人设计

### 8.1 社交本质

> **社交 = 基于需求与被需求的关系行为。**（网易内部分享）

| 洞察 | 设计含义 |
|------|----------|
| 物质需求驱动力 > 纯情感 | 先设计"利益交换"再自然衍生情感 |
| 正向+负向关系同等重要 | 敌对、竞争也是社交 |
| 玩家自带目的 | 系统与玩家目的冲突→系统被绕过或谩骂 |

### 8.2 社交价值公式（体感）

```
个人社交价值 ≈ Σ(每条关系的连接价值 × 关系强度)
```

关系越强、连接越多 → 流失成本越高。

### 8.3 社交最小单元

| 单元 | 适用 | 设计要点 |
|------|------|----------|
| **帮会/家族** | MMO、SLG | 帮战、异步协作、身份认同 |
| **固定小队** | 副本、竞技 | 一条龙文化、情感+数值双重绑定 |
| **一对一** | 结婚、师徒 | 仪式感、高门槛、高沉没成本 |

### 8.4 同步 vs 异步社交

| 类型 | 定义 | 案例 |
|------|------|------|
| **同步** | 同时同地（广义） | 组队副本、PvP |
| **异步** | A留载体，B稍后交互 | 血刀冢(《仁王2》)、悬赏墓碑(《楚留香》)、剧情弹幕(《阴阳师》) |

**异步社交双重目的**：
1. 创造社交**可能**（给你机会了，快上！）
2. 营造氛围/传达规则（"墙怪前死了好多人"）

### 8.5 多人游戏设计（陈星汉·《光·遇》GDC）

- 降低社交**表达成本**——非语言沟通
- 合作而非对抗作为默认情感基调
- 陌生人关系的**渐进信任**设计

## 九、独立开发与职业成长

### 9.1 独立开发者24条建议（E胖 McMillen 摘录）

- 做你**真正想玩**的游戏
- 范围要小——完成比完美重要
- 尽早发布测试版收集反馈
- 不要害怕奇怪的想法
- 把失败当学习，不是终点

### 9.2 独立游戏"搞钱"路径

| 路径 | 要点 |
|------|------|
| Steam海外 | 本地化、社区运营、Demo策略 |
| 国内发行 | 版号、渠道分成、长线运营预期 |
| Demo宣传 | 垂直社区投放、KOL、展会（《怎样才能让DEMO发挥最大宣传效果》） |

### 9.3 新人策划入行

| 文章 | 建议 |
|------|------|
| 《小游戏一年打工经验》 | 多做多问，小项目练全流程 |
| 《大龄新人入行分析》 | 作品集>空窗焦虑；找准细分切入点 |
| 《2022独立开发者还有出路吗》 | 有，但需差异化+社区能力 |

### 9.4 避开「抄袭陷阱」三法

| 方法 | 难度 | 做法 |
|------|------|------|
| **参与小型原创** | 最高 | GameJam、个人项目 |
| **备用设计** | 中 | 照抄版+改进版并行验证；节日活动试错 |
| **跟踪分析** | 最低 | 研究成功产品为何成功、后续如何迭代 |

**陷阱机制**：成功方案设了高基准线→改进版短期不如原版→无正反馈→躺平。

### 9.5 技术策划(TA)与工业化

- TA是美术与程序的**翻译层**
- 工业化=工具链+标准管线+可复用资产
- 小团队优先：**能跑起来 > 完美管线**
